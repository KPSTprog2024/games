# 02_RUN_MODES.md（実行モード定義）

ハーネスに内包する共通モード。
ユーザーは詳細文を毎回書かず、モード指定のみ行う。

## 共通前提（全モード）

- AGENTS / README / 既存ルールを優先
- Goal が不明瞭なら実装より先に明確化
- 最終出力は「実施 / 未実施 / リスク / 次アクション」を含む
- 毎回「残件あり/なし」を明示し、ありの場合はID付きで列挙する
- 衝突時は `00_README.md` の優先順位マトリクスに従う

---

## @pm（標準）

- 目的: 進行管理と実装をバランスする
- 出力: 計画 → 実行 → 検証 → 次アクション
- 推奨用途: 通常開発

## @fast（速度優先）

- 目的: 最小価値を早く出す
- 制約: 破壊的変更・大規模リファクタはしない
- 出力: 最短で完了したタスク + 次の高優先3件

## @safe（品質優先）

- 目的: レビュー通過率・再現性を優先
- 制約: 受け入れ基準と検証手順を先に明示
- 出力: 変更理由、代替案、検証根拠

## @plan（計画のみ）

- 目的: 実装せず設計/計画を固める
- 制約: ファイル変更を行わない
- 出力: 実行計画、リスク、判断待ち項目

---

## 推奨起動文

- `@pm 目標: <goal>`
- `@pm @safe 目標: <goal>`
- `@pm @plan 目標: <goal>`

---

## モード別ミニチェックリスト（運用短縮版）

### @pm

- [ ] Goal 1行
- [ ] Backlog ID で対象を特定
- [ ] 実行後に `✅/⏳/🧊` を更新

### @fast

- [ ] 破壊的変更をしない
- [ ] 最短完了タスクを1つ先に終える
- [ ] 「次の高優先3件」を明記

### @safe

- [ ] 受け入れ基準を先に記述
- [ ] 検証手順を実行前に定義
- [ ] 代替案を最低1つ添える

### @plan

- [ ] 実装しないことを宣言
- [ ] 依存関係と判断待ち項目を列挙
- [ ] 次ターンの着手条件を明記
