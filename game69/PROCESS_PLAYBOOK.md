# PROCESS_PLAYBOOK: game69

## フェーズ定義
1. Discovery
   - ユーザー要求を機能要求/非機能要求に分解。
2. Planning
   - UI操作フロー、データモデル、受け入れ基準を確定。
3. Build
   - Canvas描画、線分管理、補間・アニメーション実装。
4. Validate
   - 受け入れテスト、パフォーマンステスト、操作性レビュー。
5. Release/Close
   - ドキュメント更新、残課題の次回移送。

## 意思決定ルール
- 重要仕様は Decision Log に記録し、理由と棄却案を残す。
- 数式処理に関する変更は、視覚結果（再現スクリーンショット）を必須添付。

## 品質ルール
- 入力操作は取り消し可能（Undo）を最低1段階提供。
- 主要操作は3クリック以内で実行できること。
- 生成本数が増えてもUIがフリーズしないこと（目安: 200本で1秒以内に初回描画）。
