# GAME ESSENCE (既存コード分析)

## 既存ゲームから抽出した本質
1. **コア行為は1つだけ**: リズムに合わせてジャンプ入力する。
2. **快感の源泉**: 判定成功時の連続コンボと「テンポ一致」の手応え。
3. **失敗体験**: タイミング理由が曖昧だと理不尽に感じる。
4. **上達導線**: BPM・判定幅・フィードバック表示で自己調整できる。

## 既存実装の問題（なぜ遊べないか）
- UIが学習向け機能過多で、最初の5秒で何をすべきか分かりづらい。
- 成功/失敗がゲームループ（目的・報酬・再挑戦）に整理されていない。
- プレイ中の「今どこが危険か」の視覚情報が不足。

## 高度目的（逆算）
- このゲームは「縄跳びの苦手克服を、短い反復で楽しくする」ことが目的。
- つまり、必要なのは **練習教材** ではなく **何度も挑戦したくなる短尺アクション**。
