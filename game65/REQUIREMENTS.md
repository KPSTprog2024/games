# game65 要件定義（4人反射バトル）

## 1. 目的（本質の抽出）
- 競技の本質は「不確実な開始合図に対する即応速度」と「早押しリスク管理」。
- 2人版の表面的レイアウトを踏襲せず、4人が同時参加できる対面型設計へ再構成する。

## 2. ゲーム体験要件
- プレイヤーは4人（🦊 / 🐼 / 🐸 / 🐙）固定。
- 各ラウンドは `待機(赤) -> 合図(緑) -> 勝敗確定`。
- 合図後、最初に押した1人がラウンド勝者として1点獲得。
- 合図前に押したプレイヤーはフライング。違反者以外の3人に各1点付与。
- 先に5点以上に到達したプレイヤーが優勝。

## 3. UI/操作要件
- 4つの大きいタップゾーンを2x2で表示。
- 中央パネルに状態文、スコア、履歴、操作ボタンを集約。
- 操作は `スタート` / `次ラウンド` / `リセット`。

## 4. 判定要件
- ラウンド中の入力は最初の1イベントのみ有効。
- ゲーム終了時は入力無効化し、状態文で勝者を表示。
- 履歴にはラウンド番号・勝者・要因（反応値 or フライング）を記録。

## 5. 非機能要件
- 依存ライブラリなし（単一HTML配信可能）。
- モバイルで誤スクロールしにくいよう `touch-action: manipulation` を適用。
