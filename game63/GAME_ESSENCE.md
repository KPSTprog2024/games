# game1 本質定義書

## 1. ゲームの本質（コア体験）
このゲームの本質は、**短時間だけ提示された配置情報を保持し、ルールに沿って再生する「作業記憶チャレンジ」**である。  
プレイヤーは「見た情報をどれだけ正確に思い出せるか」を試され、正答・誤答が即時に可視化される。

- 記憶対象: 数字の位置または色ブロックの位置
- 記憶負荷: レベル上昇により対象数が増える
- 緊張要素: 表示は一定時間のみで、その後は不可視
- 評価軸: 正確性（正しいセルを選べるか）

## 2. プレイヤー行動の基本ループ
1. 設定選択（表示時間 / グリッドサイズ / モード）
2. カウントダウンで集中状態へ遷移
3. 一時表示された情報を記憶
4. 非表示後にセルをタップして再現
5. 正解ならクリア、誤りならリトライ
6. 継続時は次レベルへ進み、記憶負荷が上がる

## 3. ルールの中核
### 3.1 共通
- 対象はグリッド上にランダム配置される。
- 対象の表示は `displayTime` 経過後に隠れる。
- レベルで必要対象数を制御し、基本式は **レベル + 2**。

### 3.2 数字モード
- 1 から順番にタップする必要がある。
- 空セルまたは順番違いをタップした時点で失敗。
- 正答セルは即時に可視化され、進行感が返ってくる。

### 3.3 色モード
- 表示されていた色セルをすべて見つければ成功。
- 順番制約はなく、網羅性が問われる。
- 空セルタップで失敗。

## 4. 難易度設計の骨子
- **対象数増加**: レベル上昇で記憶対象が増える。
- **盤面拡大**: 3x3 / 4x4 / 5x5 の選択で探索空間が拡大。
- **記憶時間短縮**: 0.5秒〜3秒で初見負荷を調整。
- **上限制約**: 対象数が総マス数を超える場合は進行不能とみなし設定画面に戻す。

## 5. フィードバック設計
- クリア時: 正解配置の再提示 + 賞賛文 + パーティクル演出。
- 失敗時: 正解配置を見せ、誤タップ位置に「×」を重ねる。
- どちらも「再挑戦」導線を用意し、学習ループを切らさない。

## 6. 体験価値（このゲームならでは）
- 短時間で反復可能な**記憶トレーニング体験**。
- 数字モードは「順序記憶」、色モードは「位置記憶」の側面を強調。
- 幼児〜低学年にも触れやすい、ひらがな中心のUIと直感的操作。

## 7. 非本質（入れ替え可能な要素）
以下は体験の核ではなく、変更してもゲームの本質は維持される。
- 配色・フォント・演出の見た目
- 賞賛メッセージの文言
- ボタン形状やアニメーション細部

## 8. 本質を壊す変更の基準
次の変更を行うと本作のコア体験が崩れる可能性が高い。
- 記憶フェーズ（表示→非表示）を無くす
- ランダム配置を無くして固定化する
- 成否条件を曖昧にして即時フィードバックを失う
- レベル進行による負荷上昇を無くす

---
本定義書は `game1` 実装（Selection/Countdown/Game/Clear/Retry 各シーンとクリック判定ロジック）を基に、
「プレイヤーが何を試され、何を学習するゲームか」を抽出して整理したものである。
