# game63 プロジェクトマネージャー運用書

## 0. この文書の目的
`game63` にある既存成果物（`GAME_ESSENCE.md` と `outputs/`）を前提として、
**「5歳の暗記体験を、段階的に10歳相当へ底上げする“圧倒的バージョンアップ版ゲーム”を実装まで落とすための、PM実行ルール**」を定義する。

---

## 1. 最重要原則（不変）
1. 本質ループ（表示→非表示→再生）を守る。
2. 5歳児の成功体験を毎セッションで作る。
3. 「難しくする」だけでなく、記憶方略（チャンク化・順序化・想起間隔）を育てる。
4. 1回 1〜3分、日次 5〜10分で回る運用を崩さない。
5. すべての意思決定は記録し、次実行で再利用できるようにする。

---

## 2. 参照優先順位（毎回この順で読む）
1. `NEXT_PROMPT_LOG.md` の **LATEST** セクション（最優先）
2. `outputs/07_SUPER_UPGRADE_MASTER_PLAN.md`
3. `outputs/03_THREE_GAME_CONCEPTS.md`
4. `outputs/02_PERSONA_PANEL_DISCUSSION.md`
5. `GAME_ESSENCE.md`

> 迷ったら「本質維持」と「5歳の継続性」を優先する。

---

## 3. PMの実行サイクル（1スプリント）

### Phase A: 方針確定
- 最新プロンプトから今回の目的・DoD・制約を抽出
- 今回スプリントで「やらないこと」も明記

### Phase B: 仕様化
- `outputs/04_UPDATE_BLUEPRINT_TEMPLATE.md` をベースに、対象機能の仕様を埋める
- KPI（正答率・継続率・想起改善）を最低1つ紐づける

### Phase C: 実装
- MVP最小差分で導入
- UI文言は短文ひらがな優先
- 失敗時は次の一手ヒントを必ず返す

### Phase D: 検証
- 「2連敗で緩和・2連勝で微増」が守られているか確認
- 中断→復帰動線を確認

### Phase E: 引き継ぎ
- `NEXT_PROMPT_LOG.md` へ新規追記（上書き禁止）
- LATESTポインタを新IDへ更新

---

## 4. スコープ戦略（圧倒的バージョンアップ）

### Sprint 1: 即効性（案1中心）
- チャンク化支援の導入
- 成功体験の密度向上

### Sprint 2: 定着化（案3中心）
- 遅延再想起（Spaced Retrieval）
- セッション終盤の再来問題

### Sprint 3: 上位強化（案2中心）
- 順序記憶 + リズム補助
- 個別テンポ最適化の入口

---

## 5. PMチェックリスト（毎回）
- [ ] 最新プロンプトを読んだ
- [ ] 本質ループを壊していない
- [ ] 5歳UI（短文・ひらがな）を維持
- [ ] 失敗時ヒントを実装/設計した
- [ ] 次回実行用プロンプトを追記した
- [ ] LATESTポインタを更新した

---

## 6. 運用ルール（厳守）
- `NEXT_PROMPT_LOG.md` は **追記のみ**（既存プロンプト本文の改変禁止）。
- 各プロンプトに `Prompt-ID`, `Created-At`, `Goal`, `Scope`, `DoD`, `Next` を必ず含める。
- 次回実行時は、必ず LATEST の `Prompt-ID` を読み、そこから開始する。

---

## 7. 想定するPMペルソナ
- 役割: 仕様・進行・優先順位・引き継ぎを担う
- 行動特性:
  - 仕様を小さく分割する
  - 1スプリント1勝ち筋に集中する
  - 「成果が見える最短導線」を選ぶ
