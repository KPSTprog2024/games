# ファシリテーション設計（ディレクター進行案）

## 1. ミッション定義

**目的**: `GAME_ESSENCE.md` のコア（短時間提示→非表示→再生）を壊さず、5歳児の暗記能力を段階的に10歳相当へ近づけるゲーム体験を設計する。

### 成功条件（合意KPI）
- **保持**: 見せた情報を保持できる量（チャンク数）が増える
- **想起**: 非表示後の再生正答率が上がる
- **転移**: ゲーム外（絵本語彙、日常手順）に記憶戦略が持ち出せる
- **継続**: 5歳児が嫌がらず継続できる（短時間達成感）

## 2. 進行ロール

- **ディレクター（進行）**: 議題管理・意思決定・優先順位付け
- **幼児学習デザイナー**: 発達段階に合う課題負荷を担保
- **認知心理アドバイザー**: 作業記憶・想起強化の妥当性を確認
- **UX/ゲームデザイナー**: 気持ちよいループと演出設計
- **保護者代表**: 家庭運用（時間・負担・安全）を評価
- **実装リード**: 既存 game63 への実装可能性を評価

## 3. 議論の型（60分想定）

1. **前提合わせ（10分）**
   - 本質を固定: 「表示→非表示→再生」
   - 禁止変更確認: 本質破壊に該当する案は除外
2. **発散（20分）**
   - 記憶戦略の足場（語呂・ストーリー化・リズム）を列挙
3. **収束（20分）**
   - 5軸評価（効果・実装難度・継続性・安全性・拡張性）
4. **決定（10分）**
   - 上位3案選定
   - 各案のMVP範囲を確定

## 4. 意思決定ルーブリック（5段階）

- **認知効果**: 暗記力向上に直接効くか
- **年齢適合**: 5歳でも理解・操作できるか
- **楽しさ**: 成功体験と再挑戦導線があるか
- **保護者運用**: 1日5〜10分で回せるか
- **実装適合**: 既存構造を活かして段階導入できるか

## 5. 成果物フォーマット（必須）

各案は以下を必ず記載する。
1. ねらう認知スキル
2. 30秒〜3分の基本プレイサイクル
3. 難易度カーブ（5歳→10歳相当）
4. 失敗時フィードバック
5. 保護者介入ポイント
6. 実装フェーズ（MVP / v1 / v2）
