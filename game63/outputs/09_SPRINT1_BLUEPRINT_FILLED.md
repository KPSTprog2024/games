# Sprint 1 青写真（記入版）

## 0. 版情報
- バージョン: v0.1 (Sprint1-MVP)
- 作成日: 2026-04-15
- 対象案: 案1（おぼえわけ探検隊）
- 責任者: PM

## 1. 問題定義
- 現状の学習課題:
  - 「どこをどう覚えるか」が子ども側で言語化されず、当てずっぽう再生が起きやすい。
- 対象ユーザー（年齢/前提）:
  - 5歳中心（ひらがな可、長文指示は苦手）
- 成果指標（例: 正答率、保持時間、継続日数）:
  - 初回3セッション平均正答率
  - 3日継続率

## 2. 体験ゴール
- 2-1. 子ども体験ゴール:
  - 「なかまで おぼえる」が自然にできる。
- 2-2. 保護者体験ゴール:
  - 5〜10分で見守れる。
- 2-3. 教育的ゴール:
  - チャンク化（まとまりで覚える）を獲得する。

## 3. コア体験整合チェック
- [x] 表示→非表示→再生が維持されている
- [x] ランダム性が維持されている
- [x] 成否が即時フィードバックされる
- [x] 難易度上昇が機能している

## 4. 機能仕様（MVP）
- 4-1. 新規機能:
  - 色チャンクハイライト
  - 失敗時ヒント（仲間選択提案）
  - 2連敗緩和 / 2連勝微増
  - 最小計測イベント実装
- 4-2. 既存機能変更:
  - 失敗ダイアログに1行ヒントを追加
  - レベル調整を連勝連敗ベースへ補強
- 4-3. UI文言（5歳可読）:
  - 「いろの なかまで おぼえよう」
  - 「ちょっと やさしく したよ」
  - 「つぎは あかから いこう」
- 4-4. 音/演出:
  - 成功時に軽いキラ演出（テンポ優先で短時間）

## 5. 難易度設計
- 初期レベル:
  - 色2チャンク、対象数少なめ
- 中間レベル:
  - 対象数増加、チャンク境界がやや曖昧
- 上位レベル:
  - 対象数増加と表示時間微短縮
- 自動調整ルール:
  - 2連敗で緩和
  - 2連勝で微増

## 6. 失敗時導線
- エラー演出方針:
  - 否定を強めず「次の手」を提示する
- リトライ導線:
  - 1タップで再挑戦
- ヒント付与条件:
  - 失敗時は常に1つ表示

## 7. 計測・分析
- 必須イベント:
  - `round_start`
  - `memorize_phase_end`
  - `recall_success`
  - `recall_fail`
  - `hint_shown`
  - `difficulty_adjusted`
  - `session_end`
- ダッシュボード指標:
  - 初回3セッション平均正答率
  - ヒント表示後3ラウンドの正答率
  - 3日継続率
- 改善判断閾値:
  - 正答率がベースライン比 +10%以上で継続
  - 継続率低下時は難易度増加幅を縮小

## 8. リスクと対策
- 過負荷リスク:
  - 2連敗で緩和を強制
- 飽きリスク:
  - 成功演出を短く毎回返す
- 保護者負担リスク:
  - 1セッション1〜3分を固定

## 9. リリース計画
- フェーズ1（MVP）:
  - 色チャンク、失敗ヒント、難易度調整、計測
- フェーズ2:
  - 形状チャンク追加（案1 v1）
- フェーズ3:
  - チャンク自動推定（案1 v2）

## 10. 受け入れ条件（DoD）
- [ ] 5歳モニタで基本操作が自走
- [ ] 3日継続率がベースライン以上
- [ ] 想起指標（ヒント後正答率）が改善
