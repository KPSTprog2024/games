# Sprint 1 タスク分解（案1: チャンク化支援MVP）

- 作成日: 2026-04-15
- 対象スプリント: Sprint 1（MVP）
- 参照: `outputs/07_SUPER_UPGRADE_MASTER_PLAN.md`, `outputs/03_THREE_GAME_CONCEPTS.md`, `GAME_ESSENCE.md`

## 1. スプリントゴール

**ゴール:**
- 「表示→非表示→再生」の本質ループを維持したまま、
  色チャンクを使って「覚え方」を提示し、初回成功率と継続意欲を上げる。

**今回やらないこと（非スコープ）:**
- 形状アイコン導入（v1対象）
- チャンク自動推定（v2対象）
- セッション内遅延再想起（Sprint 2対象）
- リズム入力（Sprint 3対象）

---

## 2. 実装タスク一覧（優先度つき）

### P0（必須: MVP成立条件）

1. **色チャンクハイライトの表示仕様を実装**
   - 表示フェーズで「同じ色は仲間」を視覚的にわかるようにする。
   - 例: 同色セルに同一の外枠/ふわっとしたグループハイライト。
   - 受け入れ条件:
     - 5歳でも見た瞬間に「まとまり」が認識できる。
     - 既存のランダム配置ロジックを壊さない。

2. **失敗時ヒント（仲間の提案）を実装**
   - ミス時に短文ひらがなで次の一手を提示。
   - 表示文例:
     - 「つぎは あかのなかま から おぼえよう」
     - 「さいしょは 1つのいろ だけ みよう」
   - 受け入れ条件:
     - 毎失敗で必ず1つヒントが出る。
     - 文言は短文・ひらがな中心。

3. **難易度自動調整（2連敗緩和 / 2連勝微増）**
   - 2連敗: 対象数-1 または 表示時間+0.5秒（下限/上限あり）
   - 2連勝: 対象数+1 または 表示時間-0.2秒（急増させない）
   - 受け入れ条件:
     - 連敗・連勝の判定がセッション内で一貫。
     - 調整結果がUIに短く表示される（例:「ちょっと やさしくしたよ」）。

4. **最小計測イベント実装**
   - 必須イベント:
     - `round_start`
     - `memorize_phase_end`
     - `recall_success`
     - `recall_fail`
     - `hint_shown`
     - `difficulty_adjusted`
     - `session_end`
   - 受け入れ条件:
     - イベントに `timestamp`, `level`, `mode`, `result`（該当時）を含む。
     - ローカル保存（JSON）でセッション単位に追跡可能。

### P1（推奨: MVP品質向上）

5. **振り返り1タップ導線の追加**
   - ラウンド終了時: 「どのなかまで おぼえた？」
   - 選択肢（1タップ）: あか / あお / きいろ / ぜんぶ
   - 受け入れ条件:
     - 入力に3秒以上かからない。
     - 未回答でも進行できる。

6. **成功演出の軽量強化**
   - チャンク単位で小さな達成演出（きらっ）
   - 受け入れ条件:
     - プレイテンポ（1〜3分）を悪化させない。

### P2（余力）

7. **保護者向けミニ要約の生成**
   - セッション末に1行表示:
     - 「きょうは 2いろのなかまで おぼえられたよ」

---

## 3. 受け入れ条件（テスト観点）

## 3.1 体験・UI
- [ ] 主要文言が短文ひらがな中心。
- [ ] 失敗時に必ず「次に何をすればよいか」が1ステップで示される。
- [ ] 1セッション1〜3分で完了可能。

## 3.2 本質整合
- [ ] 表示→非表示→再生の順序を維持。
- [ ] 記憶対象のランダム配置を維持。
- [ ] 成否の即時フィードバックを維持。

## 3.3 難易度調整
- [ ] 2連敗で緩和が発動。
- [ ] 2連勝で微増が発動。
- [ ] 調整が過激にならない（1回変化量は小さい）。

## 3.4 計測
- [ ] 必須7イベントが欠損なく記録される。
- [ ] セッション終了時に成功率と失敗率を再計算できる。

---

## 4. 開発順序（最短着手プラン）

1. イベントスキーマ定義（P0-4）
2. 難易度調整ロジック実装（P0-3）
3. 失敗時ヒント実装（P0-2）
4. 色チャンクハイライト実装（P0-1）
5. E2Eで1セッション通し確認（受け入れ条件チェック）
6. 余力があればP1へ

## 5. 次スプリントへの引き継ぎメモ

- Sprint 2で導入する遅延再想起に備え、`round_id` と `source_round_id` をイベントに持たせると拡張が容易。
- チャンク学習の有効性は「ヒント表示後3ラウンドの正答率差」でまず検証する。
- 失敗時ヒント文はA/B可能なキー管理（`hint_variant`）にしておく。
