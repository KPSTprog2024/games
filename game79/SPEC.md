# game79 仕様書: 角度拡散エコーお絵描きアプリ

## 1. 目的

`game79` は、`game27` の「描いた線が時間差でエコーとして残る」体験をベースにしつつ、既存の一方向エコーを拡張し、エコーの角度が時間・世代・設定に応じて少しずつ変化して、最終的に三方向・五方向へ扇状に広がっていくお絵描きアプリとして新規実装する。

既存の `game27` には変更を加えず、`game79` ディレクトリ内に独立したアプリとして作成する。

## 2. コンセプト

ユーザーが画面に線を描くと、元の線から遅れて複数のエコーが発生する。エコーは単に左後方など一方向へ流れるのではなく、中心方向を基準に左右へ少しずつ角度を変えながら分岐し、時間経過とともに「1本の残響」から「3方向の残響」、さらに「5方向の残響」へ広がる。

見た目の方向性は、光の尾、音波の反射、手持ち花火の火花、花火の余韻、万華鏡のような描画体験を目指す。単に線が増えるだけではなく、中心から周辺へ広がる、螺旋状に巻き上がる、竜巻のように立ちのぼる、波紋のように外へ伝わるなど、子どもが「次はどう変わるのか」を見たくなる発展性を持たせる。

## 3. 対象ユーザー

- タブレットやスマートフォンで気軽に抽象画を描きたいユーザー。
- Apple Pencil やスタイラスで、動きのある線を楽しみたいユーザー。
- 子どもでも迷わず使える、シンプルだが視覚的に派手な描画アプリを求めるユーザー。
- `game27` のエコー描画が好きで、より広がりのある表現を試したいユーザー。

## 4. 非目標

- `game27` の既存ファイルを変更しない。
- レイヤー編集、選択範囲、Undo/Redo など本格ペイントソフト機能は初期版では実装しない。
- 3D エンジンや WebGL 前提の実装にはしない。初期版は Canvas 2D で完結させる。
- 画像アップロード、クラウド保存、SNS 投稿連携は初期版では扱わない。

## 5. 主要体験

1. ユーザーが指、マウス、ペンでキャンバスに線を描く。
2. 描画直後は、元線の後方に一方向のエコーが現れる。
3. エコー世代が進むにつれて、進行方向の角度が少しずつ左右へずれる。
4. 設定された拡散段階に達すると、エコーが 3 方向へ分岐する。
5. さらに時間またはエコー世代が進むと、5 方向へ分岐する。
6. 古いエコーは透明度、サイズ、ぼかし、色相を変えながら自然に消える。
7. ユーザーは設定パネルから、広がり方、角度、色、残響量などを調整できる。

## 6. 機能要件

### 6.1 描画入力

- Pointer Events を使用し、マウス、タッチ、スタイラスを統一的に扱う。
- Apple Pencil など pressure 値を持つ入力では、筆圧を線幅または不透明度に反映できる。
- 入力点は滑らかに補間し、速く描いたときでも線が途切れにくいようにする。
- キャンバス上での描画中はページスクロールや長押しメニューを抑制する。

### 6.2 ベースストローク

- ユーザーが直接描いた線を「ベースストローク」と呼ぶ。
- ベースストロークはエコー生成の元データとして保持する。
- ベースストロークは表示上もっとも濃く、もっとも鮮明に描画する。

### 6.3 エコー生成

- エコーは一定間隔でベースストロークまたは直近のストローク状態から生成する。
- エコーには世代番号 `echoIndex` を付与する。
- エコーの最大数を設定できる。
- 最大数を超えた古いエコーは削除する。
- エコー生成間隔は UI から調整可能にする。

### 6.4 角度拡散

- `game79` の中核機能として、エコーの移動方向を角度で管理する。
- 基準方向 `baseAngleDeg` を持ち、初期状態では一方向にエコーが流れる。
- 世代が進むほど、エコーは `spreadAngleDeg` の範囲内で左右へ広がる。
- 拡散は急に切り替わるのではなく、`spreadGrowth` によって滑らかに増加する。
- 角度の計算は、極座標の方向ベクトルを Canvas 座標の `x`, `y` 変位へ変換して行う。

### 6.5 方向数の段階変化

- エコーの方向数は `branchCount` として扱う。
- 初期状態は 1 方向。
- 条件を満たすと 3 方向へ広がる。
- さらに条件を満たすと 5 方向へ広がる。
- 初期版の推奨条件はエコー世代ベースとする。
  - `echoIndex < branch3Start`: 1 方向。
  - `branch3Start <= echoIndex < branch5Start`: 3 方向。
  - `branch5Start <= echoIndex`: 5 方向。
- 将来的に時間ベース、筆速ベース、音量ベースなどへ拡張できるよう、分岐数決定ロジックは独立関数にする。

### 6.6 方向配置

- 3 方向の場合は、中心方向、左方向、右方向の 3 本を描画する。
- 5 方向の場合は、中心方向、弱い左方向、弱い右方向、強い左方向、強い右方向の 5 本を描画する。
- 方向ごとの角度オフセットは、中心を 0 とした対称配置にする。
- 例:
  - 1 方向: `[0]`
  - 3 方向: `[0, -0.5, 0.5] * spreadAngleDeg`
  - 5 方向: `[0, -0.5, 0.5, -1, 1] * spreadAngleDeg`
- 描画順は外側方向から中心方向へ描くことで、中心の線を視覚的に残しやすくする。

### 6.7 拡散モード

初期版では以下の拡散モードを用意する。

#### 6.7.1 グローイング

- 世代が進むほど角度が徐々に広がる標準モード。
- 最も直感的な初期プリセットとする。

#### 6.7.2 パルス

- 角度が周期的に開いたり閉じたりする。
- `pulseFrequency` と `pulseDepth` を設定値として持つ。

#### 6.7.3 スパイラル

- 世代ごとに基準方向そのものが少しずつ回転する。
- 花火や銀河のような視覚効果を作る。

#### 6.7.4 ミラー

- 描いた方向に対して左右対称のエコーを強調する。
- 子ども向けの万華鏡表現として使う。

### 6.8 色と透明度

- ベース色を設定できる。
- エコー世代ごとに透明度を減衰させる。
- エコー世代ごとに色相をずらすレインボーモードを用意する。
- 分岐方向ごとにわずかに色相を変え、方向の違いがわかるようにする。
- グラデーションモードでは、ベース色から終点色へ世代に応じて補間する。

### 6.9 線幅・ぼかし・縮尺

- 線幅はベースストロークの筆圧と基本線幅から決定する。
- エコーは世代に応じて線幅を細くする、または少し太くして光の尾のように見せる設定を選べる。
- エコーごとの縮尺 `scalePerEcho` を設定できる。
- エコーごとのぼかし `blurPerEcho` を設定できる。
- ぼかしはパフォーマンスに影響するため、低性能時には自動的に弱められる設計にする。

### 6.10 広がりを意識した線の装飾

三方向・五方向の分岐は基本表現とし、その上に「広がり方そのものが楽しい」装飾を重ねられるようにする。初期実装ではすべてを必須にせず、プリセットや将来拡張で段階的に追加できる設計にする。

#### 6.10.1 火花粒子

- 手持ち花火のように、エコーの先端や曲がり角から小さな粒子を散らす。
- 粒子は線の方向から外側へランダムに飛び、短時間で消える。
- 3 方向・5 方向の外側ブランチほど粒子量を増やし、広がりを強調する。
- 粒子の色は本線より明るくし、白、黄、オレンジ、ピンク、水色を混ぜられるようにする。

#### 6.10.2 中心から周辺へのブルーム

- ユーザーが描いた線を中心軸として、時間差で外側へ薄い光の膜が広がる。
- エコーが 1 方向の段階では細い尾、3 方向で扇形、5 方向で花びら状に見えるようにする。
- `radialBloomStrength` と `radialBloomRadius` を設定値として持てるようにする。
- 強すぎると画面全体が白くなるため、透明度上限を設ける。

#### 6.10.3 螺旋リボン

- エコー世代ごとに角度を回転させ、線の周囲にリボンが巻きつくような補助線を描く。
- 進行方向に沿って左右交互に揺らすことで、キャンディ、銀河、魔法の杖のような印象にする。
- `spiralRotationPerEcho` に加えて、`spiralRadius` と `spiralTightness` を設定候補にする。
- 筆速が速いほど螺旋を粗く、遅いほど細かくする発展も可能にする。

#### 6.10.4 竜巻・巻き上がり

- エコーが世代ごとに回転しながら縮尺や Y 方向移動を変え、竜巻のような巻き上がりを表現する。
- 下側は太く、上側または奥側に向かうほど細く・薄くする。
- 5 方向の外側ブランチを少し遅らせて描くと、渦の奥行きが出る。
- 子ども向けプリセットでは「たつまき」「まほうのうず」といった名前で選べるようにする。

#### 6.10.5 波紋リング

- 描画点またはストロークの節目から、円形または楕円形の波紋を外側へ広げる。
- 波紋はベースストロークを邪魔しないよう、細線・低透明度で描く。
- タップや短い線では丸い波紋、長い線では線に沿った連続波紋にする。
- `rippleInterval`, `rippleRadiusGrowth`, `rippleAlphaDecay` を設定候補にする。

#### 6.10.6 星・花びら・小さな図形スタンプ

- エコーの先端や外側ブランチに、小さな星、丸、花びら、火花形のスタンプを出す。
- 図形はランダムではなく、ストローク方向とブランチ方向に沿って配置し、描いた線から自然に生まれたように見せる。
- スタンプ密度は低めを初期値にし、画面がうるさくなりすぎないようにする。
- 将来的には「ほし」「はなび」「あわ」「ゆき」など子どもが選びやすいテーマ名にできる。

#### 6.10.7 装飾の安全弁

- 装飾は個別にオン・オフできる。
- パフォーマンス低下時は、粒子数、スタンプ密度、波紋数、ぼかしを順に減らす。
- 激しい点滅や高速明滅は避け、子どもが長く見ても疲れにくい表現にする。

### 6.12 プリセット

初期版では以下のプリセットを用意する。

#### 6.12.1 ソフトファン

- ゆっくり 3 方向へ広がる。
- 子どもでも扱いやすい標準設定。

#### 6.12.2 レインボーブルーム

- 5 方向へ大きく広がり、色相が大きく変化する。
- 派手な視覚効果を重視する。

#### 6.12.3 スパイラルエコー

- 基準方向が世代ごとに回転する。
- 円運動や渦巻きの表現を楽しめる。

#### 6.12.4 カームリップル

- 拡散角度は小さく、透明度の減衰も緩やか。
- 水面の波紋のような落ち着いた表現にする。

#### 6.12.5 スターシャワー

- 早い段階で 5 方向になり、外側方向の線が強調される。
- 花火や流星群のような印象にする。

#### 6.12.6 てもち花火

- 線の先端から火花粒子が外側へ散る。
- 3 方向・5 方向に広がるほど、外側ブランチの火花が増える。
- 子どもが短い線を描いても、ぱちぱち光る反応が返るようにする。

#### 6.12.7 うずまきリボン

- 螺旋リボンを有効にし、線の周囲に回転する飾りを付ける。
- 方向分岐と組み合わせて、魔法の渦や銀河のように見せる。

#### 6.12.8 たつまき

- 竜巻・巻き上がり表現を有効にし、世代が進むほど回転と縮尺変化を強める。
- 上にのぼる、または奥へ吸い込まれるような動きを作る。

#### 6.12.9 みずのはもん

- 波紋リングを有効にし、描画点から円形の波が外へ広がる。
- 色と透明度を控えめにし、落ち着いた水面のような体験にする。

## 7. 画面仕様

### 7.1 レイアウト

- 全画面キャンバスを基本とする。
- 右上または右側に設定パネルを表示する。
- モバイルでは設定パネルを折りたたみ式にする。
- パフォーマンス表示は小さく、描画の邪魔にならない位置に置く。

### 7.2 操作

- ドラッグまたはペン入力で描画する。
- 「クリア」でキャンバス上のベースストロークとエコーをすべて消す。
- プリセットを押すと、関連設定が即時反映される。
- スライダー変更中も描画ループを止めず、リアルタイムに見た目を変える。

### 7.3 レスポンシブ対応

- スマートフォン縦向き、スマートフォン横向き、タブレット、デスクトップに対応する。
- 画面回転時はキャンバスサイズを再計算する。
- `devicePixelRatio` を考慮し、高解像度画面でも線がぼやけすぎないようにする。

## 9. 角度計算仕様

### 9.1 分岐数

```ts
function getBranchCount(echoIndex, settings) {
  if (echoIndex >= settings.branch5Start) return 5;
  if (echoIndex >= settings.branch3Start) return 3;
  return 1;
}
```

### 9.2 拡散率

```ts
function getSpreadRatio(echoIndex, settings) {
  return Math.min(1, echoIndex * settings.spreadGrowth);
}
```

### 9.3 角度オフセット

```ts
function getBranchOffsets(branchCount) {
  if (branchCount === 1) return [0];
  if (branchCount === 3) return [0, -0.5, 0.5];
  return [0, -0.5, 0.5, -1, 1];
}
```

### 9.4 実角度

```ts
angleDeg = baseAngleDeg + spiralOffsetDeg + branchOffset * spreadAngleDeg * spreadRatio;
```

### 9.5 移動量

```ts
const rad = angleDeg * Math.PI / 180;
const distance = echoDistance * echoIndex;
const dx = Math.cos(rad) * distance;
const dy = Math.sin(rad) * distance;
```

Canvas の Y 軸は下方向が正であるため、UI 上の角度表記と内部計算の向きは実装時に明記する。

## 10. パフォーマンス要件

- 目標は一般的なタブレットで 60 FPS。
- 最低許容ラインは 30 FPS。
- エコー数、分岐数、ぼかし量が増えるほど描画負荷が高くなるため、以下の自動軽量化を検討する。
  - FPS が一定時間 45 未満なら `blurPerEcho` を弱める。
  - それでも改善しなければ最大エコー数を一時的に減らす。
  - 5 方向描画時のみ古いエコーの描画間引きを行う。
- 古いエコーから順に削除し、無制限に配列を増やさない。

## 11. アクセシビリティ

- ボタンやスライダーには意味のあるラベルを付ける。
- 色だけに依存せず、方向数や拡散量は数値表示でも確認できるようにする。
- 激しい点滅は避ける。
- 背景効果は「なし」を選べるようにする。
- 操作に必須な UI はキーボードでも変更できる標準フォーム要素を使う。

## 13. 受け入れ基準

- `game27` 配下のファイルに変更がない。
- `game79/index.html` をブラウザで開くとアプリが起動する。
- キャンバスに描画できる。
- エコーが最初は 1 方向に流れる。
- 設定された世代以降に 3 方向へ広がる。
- 設定された世代以降に 5 方向へ広がる。
- 拡散角度を変更すると、広がり方がリアルタイムに変化する。
- クリアボタンで全描画が消える。
- スマートフォン、タブレット、デスクトップで大きなレイアウト崩れがない。
- コンソールに通常操作でエラーが出ない。
- 装飾テーマを変更すると、火花、ブルーム、螺旋、竜巻、波紋などの見た目の違いが確認できる。
- 装飾をオフにすると、三方向・五方向の基本エコーだけの表示に戻る。

## 14. テスト観点

### 14.1 手動テスト

- マウスで描画できる。
- タッチで描画できる。
- ペン入力で描画できる。
- 長い線、短い点描、素早い描画で線が破綻しない。
- 1 方向、3 方向、5 方向の切り替わりが視認できる。
- 火花粒子、中心ブルーム、螺旋リボン、竜巻、波紋リングが有効なプリセットで、それぞれ異なる広がり方に見える。
- 各プリセットが期待する見た目に変化する。
- 画面回転後も描画できる。
- クリア後に古いエコーが再表示されない。

### 14.2 計算テスト

- `getBranchCount` が設定値どおり 1、3、5 を返す。
- `getBranchOffsets` が対称なオフセット配列を返す。
- `getSpreadRatio` が 0 以上 1 以下に収まる。
- 角度から `dx`, `dy` への変換が想定方向になる。

### 14.3 パフォーマンステスト

- 最大エコー数と 5 方向設定でも操作不能にならない。
- ぼかしを強めた状態で FPS 低下時の軽量化が働く。
- 長時間描画してもメモリ使用量が増え続けない。

## 15. 実装時の注意

- `game27` の体験や設計は参考にするが、ファイルは直接変更しない。
- 角度拡散の計算は描画処理から分離し、後からテストしやすい関数にする。
- UI 表示値と内部設定値の同期を明確にする。
- ぼかしや影の設定は Canvas の状態に残りやすいため、描画ごとに `save` / `restore` を使う。
- `requestAnimationFrame` による描画ループと、エコー生成タイマーの責務を分ける。
- モバイル Safari を考慮し、タッチ操作中のスクロール抑制を忘れない。
